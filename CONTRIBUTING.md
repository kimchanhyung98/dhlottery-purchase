# 기여 가이드

기여해 주셔서 감사합니다.

가능하면 변경 범위를 작게 유지해 주세요.

## 로컬 준비

```bash
npm ci
npm run lint
npm run build
```

## PR 전 확인

- 기존 이슈나 PR이 있는지 먼저 확인해 주세요.
- 큰 기능 추가나 동작 변경은 먼저 이슈로 논의해 주세요.
- 시크릿 정보는 절대 커밋하지 마세요.
- 동작이나 예제가 바뀌면 관련 문서도 함께 업데이트해 주세요.
- `src/` 변경으로 런타임 결과가 바뀌면 같은 PR에 `dist/` 변경도 포함해 주세요.
- PR 전 `npm run lint`, `npm run build`를 실행해 주세요.

## 커밋 메시지

Conventional Commits 형식을 권장하지만 필수는 아닙니다.

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`

## 코드 스타일

기존 파일 구조와 네이밍을 최대한 유지해 주세요. 불필요한 리팩터링보다는 현재 변경과 직접 관련된 범위만 수정해 주세요.
